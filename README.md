# HW2: Q-learning 與 SARSA 演算法之比較研究 (Cliff Walking)

這是一個基於強化學習（Reinforcement Learning）中經典「懸崖漫步（Cliff Walking）」環境的實作。本專案旨在透過相同的環境與參數設定，深入探討並視覺化 **Q-learning（離策略）** 與 **SARSA（同策略）** 兩種演算法在學習行為、路徑選擇與收斂特性上的根本差異。

## 🛠️ 環境與參數設定

* **環境空間**: 4 × 12 網格世界 (Gridworld)
* **起點與終點**: 起點位於左下角 `(3, 0)`，終點位於右下角 `(3, 11)`。
* **懸崖區域 (Cliff)**: 底部介於起點與終點之間的區域 `(3, 1)` 到 `(3, 10)`。
* **獎勵機制 (Reward)**:
    * 每移動一步：`-1`
    * 掉入懸崖：`-100`，並強制回到起點
    * 到達終點：回合結束
* **超參數 (Hyperparameters)**:
    * 探索率 $\epsilon$ (Epsilon): `0.1`
    * 學習率 $\alpha$ (Alpha): `0.4`
    * 折扣因子 $\gamma$ (Gamma): `0.9`
    * 訓練回合數: `500` 回合 (取 50 次實驗平均)

## 📊 結論與討論：Q-learning vs. SARSA

經過實驗與圖表分析，總結兩種演算法在本實驗中的主要差異如下：

### 1. 最終策略路徑的差異
* **Q-learning (冒險激進)**：學習到了理論上的「最短路徑」，也就是緊貼著懸崖邊緣行走。因為它是 **Off-policy**，更新時總是假設下一步會採取完美的最優動作（`max Q`），完全忽視了探索（$\epsilon$）帶來的失足風險。
* **SARSA (保守穩健)**：學習到了遠離懸崖的「安全路徑」（繞行網格最上方）。因為它是 **On-policy**，更新時基於實際採取的行動，它察覺到了在邊緣行走時有 10% 的機率會掉入懸崖，為了將整體預期懲罰降到最低，它選擇犧牲路徑長度以換取安全性。

### 2. 收斂速度與穩定性比較
* **收斂速度**：**SARSA 收斂較快**。SARSA 能在較早的回合數就學會遠離危險區域，讓平均獎勵迅速回升並穩定；而 Q-learning 需要花費較多回合在深淵中掙扎。
* **穩定性**：**SARSA 遠比 Q-learning 穩定**。在訓練過程（Online Learning）中，因為 Agent 受到 $\epsilon$-greedy 策略控制，Q-learning 的貼崖路徑會導致它頻繁觸發隨機亂走而掉入懸崖（承受 -100 懲罰），使得累積獎勵曲線波動極大且分數較低。相反地，SARSA 在安全路徑上即便隨機亂走，也只是多扣 1 分（撞牆或繞路），因此學習曲線非常平穩。

### 3. 情境選擇建議 (何時該用誰？)
這兩種演算法沒有絕對的優劣，取決於應用場景：

* **應選擇 SARSA 的情境（需要重視訓練過程的安全性）**：
    當 **「在訓練過程中犯錯的成本極高」** 時應選擇 SARSA。例如：訓練實體機器人、自動駕駛汽車，或是在不允許系統崩潰的線上環境中學習。在這些情境下，我們需要 Agent 在探索未知時保持謹慎與安全。
* **應選擇 Q-learning 的情境（追求最終部署的完美表現）**：
    當 **「訓練是在無實質成本的模擬器中進行」**，且我們的最終目標是將訓練好的模型**關閉探索（設 $\epsilon = 0$）並投入實戰**時，應選擇 Q-learning。因為它能無視過程中的風險，保證找到全局的理論最佳解（最短路徑）。
